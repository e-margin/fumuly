# 🎨 Fumuly デザイン・UI設計

*最終更新：2026-02-23*

---

## デザインコンセプト

**「封筒を開ける怖さ」を和らげる、落ち着いた安心感**

ターゲットユーザーはADHD傾向があり、督促状・債務書類など精神的負荷の高い郵便物を扱う。UIが圧迫感を与えてはいけない。余計な情報を徹底的に排除し、「今やることだけ」を見せる設計を基本とする。

### 設計原則

1. **一画面一アクション**：一度に求めることは一つだけ
2. **緊急度を色で直感的に**：文字を読まなくても重要度がわかる
3. **次の行動を必ず提示**：「で、何をすればいい？」に答え続ける
4. **電話を勧めない**：Web・コンビニ・郵送など代替手段を優先

## ブランドカラー

```css
:root {
  /* メイン */
  --color-primary:    #2C4A7C;  /* ソフトネイビー：信頼・落ち着き */
  --color-accent:     #F4845F;  /* ウォームオレンジ：行動を促す・親しみ */

  /* 背景・テキスト */
  --color-bg:         #F7F8FA;  /* オフホワイト：目に優しい */
  --color-text:       #2D2D2D;  /* チャコール：読みやすさ */
  --color-text-sub:   #757575;  /* サブテキスト */

  /* 優先度カラー */
  --color-urgent:     #E05252;  /* 🔴 緊急対応 */
  --color-action:     #F0A500;  /* 🟡 要対応 */
  --color-keep:       #52A06E;  /* 🟢 保管のみ */
  --color-ignore:     #ABABAB;  /* ⚪ 破棄可 */

  /* カード背景（薄め） */
  --color-urgent-bg:  #FDF0F0;
  --color-action-bg:  #FDF8EC;
  --color-keep-bg:    #F0F8F3;
  --color-ignore-bg:  #F5F5F5;
}
```

## 画面構成・遷移図

```
起動
  │
  ├─【初回のみ】オンボーディング（同意画面）
  │         ↓
  └─────── ホーム
              │
              ├──→ スキャン
              │       ↓
              │    カメラ起動
              │       ↓
              │    解析中（ローディング）
              │       ↓
              │    解析結果確認
              │       ↓
              │    保存 → ホームへ戻る
              │
              ├──→ 一覧
              │       ↓
              │    書類詳細
              │       ↓
              │    対応済みにする / 削除
              │
              └──→ 設定
                      ├─ 通知設定
                      └─ データ削除
```

## 各画面の要素定義

### 1. オンボーディング（初回のみ）

| 要素 | 内容 |
|------|------|
| 説明テキスト | AIに画像を送信すること・データの扱いを簡潔に説明 |
| 同意ボタン | 「同意して始める」（プライマリボタン） |
| 拒否リンク | 「同意しない」（テキストリンク） |
| 補足リンク | プライバシーポリシーへのリンク |

---

### 2. ホーム

| 要素 | 内容 |
|------|------|
| ヘッダー | アプリロゴ・未対応件数バッジ |
| 緊急カード | 🔴 未対応の緊急書類を最大3件表示 |
| 要対応カード | 🟡 要対応書類の件数サマリー |
| スキャンボタン | 画面下部固定・大きめ・アクセントカラー |
| ナビゲーション | ホーム / 一覧 / 設定（ボトムナビ） |

**設計注意**：ホームに表示するのは「今すぐ対応が必要なもの」のみ。保管・破棄可は一覧から確認。

---

### 3. スキャン

| 要素 | 内容 |
|------|------|
| カメラビュー | 全画面表示・撮影ガイド枠 |
| 撮影ボタン | 下部中央・大きく |
| 複数枚インジケーター | 何枚撮影済みか表示 |
| 解析開始ボタン | 撮影後に表示「解析する」 |
| ローディング | 「AIが読んでいます...」のアニメーション |

**設計注意**：ローディング中は「少し待ってね」的なコピーで不安を和らげる。

---

### 4. 解析結果確認

| 要素 | 内容 |
|------|------|
| 優先度バッジ | 🔴🟡🟢⚪ + ラベル（大きく表示） |
| サマリー | AIが生成した一言説明 |
| 送付元 | 機関名・企業名 |
| 期限 | 日付（期限が近い場合は赤字） |
| 金額 | 円表示（該当する場合） |
| 推奨アクション | 「次にすべきこと」をカード形式で表示 |
| 保存ボタン | 「保存する」（プライマリ） |
| 撮り直しリンク | 「撮り直す」（テキストリンク） |

---

### 5. 一覧

| 要素 | 内容 |
|------|------|
| フィルタータブ | すべて / 緊急 / 要対応 / 保管 / 破棄可 |
| ソート | 期限順 / 登録日順 |
| 書類カード | 優先度カラー・送付元・期限・対応済みフラグ |
| 対応済みトグル | カード内でスワイプまたはチェック |

---

### 6. 書類詳細

| 要素 | 内容 |
|------|------|
| 解析結果全文 | サマリー・送付元・種別・金額・期限 |
| 推奨アクション | 具体的な手順 |
| 撮影画像 | ローカルから読み込み表示 |
| 対応済みボタン | 「対応済みにする」 |
| 削除ボタン | 「この書類を削除」（破壊的操作なので確認ダイアログ） |

---

### 7. 設定

| 要素 | 内容 |
|------|------|
| 通知設定 | 期限X日前に通知（日数を選択） |
| データ削除 | 「すべてのデータを削除」（確認ダイアログ必須） |
| プライバシーポリシー | リンク |
| バージョン情報 | アプリバージョン表示 |

## UIコンポーネント定義

### 優先度カード

```
┌─────────────────────────────┐
│ 🔴 緊急対応                   │  ← カラーバー（左端）
│                              │
│ 東京電力フュエル&パワー          │  ← 送付元（Bold）
│ 電気料金 督促状                │  ← 書類種別
│                              │
│ ¥12,800        期限 3/5      │  ← 金額・期限
│                              │
│ 💡 コンビニで今すぐ支払えます    │  ← 推奨アクション
└─────────────────────────────┘
```

### ボタン定義

| 種類 | 色 | 用途 |
|------|-----|------|
| プライマリ | `--color-primary` | メインアクション |
| アクセント | `--color-accent` | スキャンボタン・CTAなど |
| テキスト | `--color-text-sub` | サブアクション・リンク |
| 破壊的 | `--color-urgent` | 削除・取り消し |

## アクセシビリティ方針

- フォントサイズ最小16px（本文）
- タップ領域最小44×44px
- カラーのみに依存しない情報伝達（アイコン・テキスト併用）
- コントラスト比 4.5:1 以上（WCAG AA準拠）
